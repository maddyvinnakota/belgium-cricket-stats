## The Core Problem — CricClubs Auth

CricClubs protects its REST API with a time-based `x-content-token` header generated by client-side JS. There is no public API key. Token expires every ~60 seconds.

**Solution**: Playwright (headless Chromium) to:
1. Navigate to `cricclubs.com/CricketBelgium` and intercept all outgoing requests to `core-prod-origin.cricclubs.com`
2. Extract `x-content-token` from intercepted headers AND from `window.__xct` (monkey-patched XHR + fetch via `addInitScript`)
3. Cache token for 90 seconds, make direct axios calls with it
4. When stale, navigate again to refresh

**Navigation mutex**: All `page.goto` calls serialised through a queue:
```js
let _navQueue = Promise.resolve();
function enqueueNav(fn) {
  const result = _navQueue.then(() => fn());
  _navQueue = result.then(() => {}, () => {});
  return result;
}
```
Token refresh must also go through this mutex. Concurrent navigations crash Playwright.

**Required headers on every axios call**:
```
x-content-token: <captured>
Accept: application/json
User-Agent: Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7)...Chrome/124.0.0.0...
Referer: https://cricclubs.com/
Origin: https://cricclubs.com
```

**Error handling**: On 4xx or `errorCode: 'SEC001'` in response body → refresh token once and retry. Never retry more than once.

## CricClubs API Endpoints

Base: `https://core-prod-origin.cricclubs.com/core/public`
All calls append `?v=4.0.536`

| Endpoint | Key Params | Notes |
|---|---|---|
| `series/list` | `leagueId, page, size` | Returns `{data:{seriesList:[...]}}` |
| `team/getPointsTable` | `clubId=LEAGUE_ID, seriesId` | Uses `clubId` not `leagueId` |
| `series/{seriesId}/matches` | `leagueId, size=1000, page=1` | Default page size is 30 — must pass 1000. T20 CUP has 156+ matches |
| `series/{seriesId}/matches` | `leagueId, teamId, status=All, year=YYYY, size=1000` | With `teamId` → per-team data with `selectedTeamPoints` |
| `league/{leagueId}/series/{seriesId}/details` | — | Series rules: winPoints, bonusPoints etc. |
| `stats/getPlayerStats` | `clubId=LEAGUE_ID, seriesId` | Player stats |

**Constants**:
```
LEAGUE_ID = 'j-x9YyxrIre5Zj_OS9HQyA'
SITE      = 'https://cricclubs.com/CricketBelgium'
```

## Critical Data Shapes

**Match object** (from `series/{seriesId}/matches`):
```json
{
  "fixtureId": "LA82bh6fwzdx2TXrTFF7jw",
  "matchDateTime": "2026-03-29T10:00:00",
  "teamOne": { "id": "...", "name": "Team A" },
  "teamTwo": { "id": "...", "name": "Team B" },
  "status": "Completed",
  "condition": "past",
  "scoreSummary": { "result": "Team B won by 10 Wickets" },
  "ground": { "name": "Ground Name" }
}
```
⚠️ Same match appears in both `completed[]` and `scheduled[]` with different `fixtureId`s.
Dedup by `date.slice(0,10) + sorted team names`. Completed wins.

**Team-match with exact points** (add `teamId` param):
```json
{ "fixtureId": "...", "selectedTeamPoints": 23, "scoreSummary": { "result": "..." } }
```
`selectedTeamPoints` = base + bonus. Even losing teams can earn bonus points.
This is the only accurate points source for T20 CUP.

**Points table team** (raw from API):
```json
{
  "encryptedTeamId": "Q-2fPuQk9HAgye6jrzn5SQ",
  "teamName": "Beveren CC",
  "teamCode": "BCC",
  "matches": 9, "won": 6, "lost": 3, "tied": 0, "noResult": 0,
  "points": 148, "netRunRate": 1.26,
  "runsScored": 1509, "runsGiven": 1341,
  "ballsFaced": 1023, "ballsBowled": 1060,
  "logo_file_path": "path/to/logo.png"
}
```
Logo: `https://static.cricclubs.com/{logo_file_path}`

## Server Caching (server.js)

Per-prefix TTLs:
```js
const TTL = {
  series: 24*60*60*1000,  // 24h
  rules:   6*60*60*1000,  //  6h
  pt:     30*60*1000,     // 30min (points table)
  ml:     30*60*1000,     // 30min (match list)
  tm:     30*60*1000,     // 30min (team matches)
  ps:     30*60*1000,     // 30min (player stats)
};
```
Also send `Cache-Control: public, max-age=N, stale-while-revalidate=2N` headers on every response.

Cache implementation:
```js
const _cache = new Map();
function cGet(k) { const e = _cache.get(k); return e && Date.now()-e.ts < ttlFor(k) ? e.d : null; }
function cSet(k, d) { _cache.set(k, { d, ts: Date.now() }); return d; }
async function cached(k, fn) { return cGet(k) ?? cSet(k, await fn()); }
```

## API Routes (server.js)

| Route | Params | Returns |
|---|---|---|
| `GET /api/series` | — | Series list |
| `GET /api/points-table` | `seriesId` | Array of groups, each with normalised teams |
| `GET /api/matches` | `seriesId` | `{ matches: [...], rules: {...} }` deduplicated, sorted by date |
| `GET /api/schedule` | `seriesId` | Same deduplicated match array (no rules) |
| `GET /api/team-matches` | `seriesId, teamId, year` | `[{fixtureId, selectedTeamPoints, result, matchDateTime}]` |
| `GET /api/player-stats` | `seriesId` | Raw API passthrough |

**Normalised team object** (exposed by `/api/points-table`):
```js
{
  rank, team, code, teamId,  // teamId = encryptedTeamId
  logo,                       // full URL with static.cricclubs.com prefix
  m, w, l, t, nr, pts, nrr,
  runsFor, runsAgst, oversFor, oversAgst,
  forDisplay, agstDisplay,   // "runs/overs" strings
  matchSchedule              // upcoming matches from points table
}
```

**Match deduplication**:
```js
const key = m => `${(m.matchDateTime??m.date??'').slice(0,10)}|${[m.teamOne?.name??'',m.teamTwo?.name??''].sort().join('|')}`;
const byKey = new Map();
for (const m of [...(raw.completed??[]), ...(raw.ongoing??[]), ...(raw.scheduled??[])]) {
  if (!byKey.has(key(m))) byKey.set(key(m), m); // completed wins
}
```

## Frontend (public/index.html)

Single file, no framework, no build step. Four tabs: **Points Table | Points Summary | Points Graph | Schedule**.

### Client localStorage Cache (stale-while-revalidate)
On cache hit: return immediately + silently refresh in background via `fetch()`.
TTLs mirror server. Key prefix: `bcs_`.

```js
async function api(path) {
  const cached = lsGet(path);
  if (cached !== null) {
    fetch(path).then(r => r.json()).then(d => lsSet(path, d)).catch(() => {});
    return cached;
  }
  const res = await fetch(path);
  const data = await res.json();
  lsSet(path, data);
  return data;
}
```

### Default Series on Load
First ONGOING series matching `/t20.cup/i`, else first ONGOING:
```js
let defIdx = allSeries.findIndex(s => s.status === 'ONGOING' && /t20.cup/i.test(s.name));
if (defIdx < 0) defIdx = allSeries.findIndex(s => s.status === 'ONGOING');
```

### Team Display
Always `${t.code} ${t.team}` (e.g. "BCC Beveren Cricket Club-1") everywhere.

### Team Colour
Seeded by team name using hash into a 10-colour palette. Consistent across sessions:
```js
const TEAM_COLORS = ['#3182ce','#805ad5','#d69e2e','#38a169','#e53e3e',
                     '#dd6b20','#00b5d8','#e91e8c','#b7791f','#2c7a7b'];
function teamColor(name) {
  let h = 0;
  for (let i = 0; i < name.length; i++) h = (Math.imul(31, h) + name.charCodeAt(i)) | 0;
  return TEAM_COLORS[Math.abs(h) % TEAM_COLORS.length];
}
```

### Tab 1 — Points Table
Standard standings: Rank, Team (logo/initials + "CODE Name"), M, W, L, T, NR, Pts, NRR, For, Against, Form (last 5 badges).
Logo fallback: coloured circle with 2-letter initials on `onerror`.
NRR: green if positive, red if negative.

### Tab 2 — Points Summary
Matrix: rows = teams, columns = M1…MN, last column = official Pts.

Each completed cell:
- **W/L/NR/T** (bold)
- **+N** exact points from `selectedTeamPoints` (falls back to base pts from rules)
- **OPP** 3-letter opponent code (small, muted)

Upcoming cells: grey background, date + opponent code.

**How exact per-game points work**:
1. Fetch points table → get all `teamId` values
2. `Promise.all` → fetch `/api/team-matches` for every team
3. Build lookup: `{ teamName → { fixtureId → selectedTeamPoints } }`
4. Match `fixtureId` from full match list to lookup → exact points per cell
5. Fall back to base pts from rules if lookup misses

**`matchResultForTeam(match, teamName)`**:
```js
function matchResultForTeam(match, teamName) {
  const r = (match.result ?? '').trim();
  if (!r) return null;
  if (/abandoned|no result/i.test(r)) return 'NR';
  if (/tie|tied/i.test(r))            return 'T';
  const wonIdx = r.toLowerCase().indexOf(' won');
  if (wonIdx > -1) {
    const winner = r.slice(0, wonIdx).trim();
    return winner === teamName ? 'W' : 'L';
  }
  return null;
}
```

**Opponent abbreviation**: uses `t.code` for known teams; for unknown names, derives from first letter of each significant word (skip: "cricket", "club", "cc", "bc"), max 3 chars, cached.

**Year for team-matches API**: find series in `allSeries` by `seriesId`, use `.year` field or extract with `/\b(20\d{2})\b/` from series name.

### Tab 3 — Points Graph
Line chart (Chart.js) showing simulated cumulative points over matches.
Uses W/L/NR from result strings + base points from rules.
Note: T20 CUP bonus points are approximated here — Summary tab has exact values.
Custom clickable legend buttons to toggle team lines.

### Tab 4 — Schedule
Per-team schedule cards (all matches sorted by date).
Completed: result string. Upcoming: date + ground.
Below: "Results Summary" with W/L/NR badge counts per team.

## T20 CUP Bonus Points — Key Insight

T20 CUP 2026 rules: `winPoints:20, lossPoints:0, tiePoints:10, abandonedPoints:10, bonusPoints:1, superLeagueBonusPoints:1`

Bonus is variable per match — a win gives 20–25 pts, a loss can give 0–5 pts bonus. Cannot be computed from W/L alone. The only accurate source is `selectedTeamPoints` from the per-team endpoint (`series/{seriesId}/matches?teamId=...`). Summing it across all matches exactly equals the official `points` total in the standings.

Standard league divisions have `bonusPoints:0, superLeagueBonusPoints:0` — cumulative is exact from base points alone.

## Series with Divisions

BCF League has divisions (Div 1, Div 2 etc.) — top-level series has a `divisions[]` array, each division has its own `seriesId`. T20 CUP and other competitions are flat (no divisions). Show a division selector only when `s.divisions.length > 0`.

## Gotchas — Do Not Get These Wrong

1. **`series/{seriesId}/matches` uses `leagueId` param. `team/getPointsTable` uses `clubId`.** Do not swap.
2. **Always pass `size:1000`** to match endpoints — default is 30, T20 CUP has 156+ matches.
3. **Deduplicate matches** (completed wins). The same match appears in both `completed[]` and `scheduled[]`.
4. **Do not use `captureViaNav` for points table** — CricClubs fires a wrong-seriesId request on page load; always use `directCall` with a fresh token.
5. **All `page.goto` calls go through the nav mutex**, including `refreshToken`.
6. **`directCall` stays internal to `scraper.js`**. `server.js` imports only named exported functions.
7. **T20 CUP graph is approximate** (bonus not computable per-match without scorecard). Only the Summary tab has exact values via `selectedTeamPoints`.
8. **Logo URLs need the static prefix** — `logo_file_path` from API is a fragment; prepend `https://static.cricclubs.com/`.
9. **Token refresh is also enqueued** — parallel API calls during refresh must not each trigger a separate navigation.
